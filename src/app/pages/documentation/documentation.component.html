Documentação

Início Rápido

Sakai é um modelo de aplicativo para Angular e é distribuído como um projeto CLI. A versão atual é Angular v17 com PrimeNG v17. Caso o CLI ainda não esteja instalado, use o comando abaixo para configurá-lo.

npm install -g @angular/cli

Assim que o CLI estiver pronto no seu sistema, extraia o conteúdo do arquivo zipado, acesse o diretório, instale as bibliotecas via npm e execute "ng serve" para rodar a aplicação no seu ambiente local.

cd sakai
npm install
ng serve

A aplicação deve rodar em http://localhost:4200/; agora você pode começar o desenvolvimento da sua aplicação.

Comandos Importantes do CLI

Os seguintes comandos derivam do CLI.

Execute 'ng serve' para um servidor de desenvolvimento. Acesse `http://localhost:4200/`. A aplicação recarregará automaticamente se você alterar qualquer um dos arquivos de origem.

Execute 'ng generate component nome-do-componente' para gerar um novo componente. Você também pode usar `ng generate directive/pipe/service/class/module`.

Execute 'ng build' para construir o projeto. Os artefatos da construção serão armazenados no diretório `dist/`. Use a bandeira `-prod` para uma construção de produção.

Execute 'ng test' para executar os testes de unidade via [Karma](https://karma-runner.github.io).

Execute 'ng e2e' para executar os testes end-to-end via [Protractor](http://www.protractortest.org/).

Execute 'ng help' para mais opções.

Estrutura

Sakai consiste em 3 partes principais: o layout da aplicação, assets do layout e assets do tema do componente PrimeNG. O layout está localizado dentro da pasta src/app/layout, e os assets estão na pasta src/assets/layout.

Configuração Padrão

A configuração inicial do layout pode ser definida no componente principal do aplicativo ao injetar o LayoutService; este passo é opcional e apenas necessário ao personalizar as configurações padrão. Observe que theme e scale não são reativos, pois o tema é configurado fora do Angular em index.html por padrão e a escala inicial é definida com $scale em layout.scss. Quando o tema ou a escala padrão são alterados nesses arquivos inicialmente, é necessário configurar o serviço de layout com os valores correspondentes para evitar problemas de sincronização.

import { Component, OnInit } from '@angular/core';
import { PrimeNGConfig } from 'primeng/api';
import { LayoutService, AppConfig } from './layout/service/app.layout.service';

@Component({
    selector: 'app-root',
    templateUrl: './app.component.html'
})
export class AppComponent implements OnInit {

    constructor(private primengConfig: PrimeNGConfig, private layoutService: LayoutService) { }

    ngOnInit(): void {
        this.primengConfig.ripple = true;       //habilita a funcionalidade básica de ripple

        //configuração opcional com a configuração padrão
        const config: AppConfig = {
            ripple: false,                      //liga e desliga o ripple
            inputStyle: 'outlined',             //estilo padrão para elementos de input
            menuMode: 'static',                 //modo de layout do menu, valores válidos são "static" e "overlay"
            colorScheme: 'light',               //esquema de cores do template, valores válidos são "light" e "dark"
            theme: 'lara-light-indigo',         //tema padrão do componente para PrimeNG
            scale: 14                           //tamanho da fonte do corpo para dimensionar toda a aplicação
        };
        this.layoutService.config.set(config);
    }

}
Menu

O menu é um componente separado definido no arquivo src/app/layout/app.menu.component.ts e baseado na API MenuModel do PrimeNG. Para definir os itens do menu, acesse este arquivo e defina seu próprio modelo como uma estrutura aninhada.

import { OnInit } from '@angular/core';
import { Component } from '@angular/core';

@Component({
    selector: 'app-menu',
    templateUrl: './app.menu.component.html'
})
export class AppMenuComponent implements OnInit {

    model: any[] = [];

    ngOnInit() {
        this.model = [
            {
                label: 'Início',
                items: [
                    {
                        label: 'Painel de Controle',
                        icon: 'pi pi-fw pi-home',
                        routerLink: ['/']
                    }
                ]
            },
            //...
        ];
    }
}

Integração com Projetos Angular CLI Existentes

A estrutura do Sakai é projetada de maneira modular para que possa ser facilmente integrada ao seu aplicativo existente. Criamos um tutorial curto com detalhes.

Tema

O Sakai fornece 34 temas PrimeNG prontos para uso. A configuração de um tema é simples, incluindo o CSS do tema no seu pacote, localizado dentro da pasta assets/layout/styles/theme/ como assets/layout/styles/theme/lara-light-indigo/theme.css.

Outra alternativa seria criar bundles dinâmicos, consulte o video tutorial for an example.
